# 交互设计规范

本文档描述 RBAC 系统的交互设计规范和 UX 最佳实践。

---

## 通用交互原则

### 1. 反馈及时性

| 操作类型 | 反馈时间 | 反馈方式 |
|---------|---------|---------|
| **点击按钮** | < 100ms | 视觉反馈（按下效果） |
| **表单验证** | < 300ms | 错误提示显示 |
| **API 请求** | < 300ms | Loading 状态 |
| **页面切换** | < 500ms | 路由切换动画 |

### 2. 操作可撤销

| 操作 | 撤销方式 |
|------|---------|
| **删除数据** | Toast 提示「已删除」+ 「撤销」按钮 (5秒内) |
| **批量操作** | 确认弹窗 + 可取消 |
| **表单填写** | 自动保存草稿 |

### 3. 状态可见性

- 所有操作必须有明确的成功/失败反馈
- Loading 状态必须显示（spinner 或骨架屏）
- 禁用状态必须有视觉区分和原因说明

## 表格交互

### 1. 行选择

```
交互规则:
- 点击行 → 高亮选中行，显示蓝色左边框
- 点击复选框 → 勾选/取消勾选
- Ctrl+点击 → 多选（桌面端）
- Shift+点击 → 范围选择（桌面端）

视觉反馈:
- 悬停: 行背景变为浅灰
- 选中: 行背景变为浅蓝，左边框 3px 蓝色
- 半选: 复选框显示横线状态
```

### 2. 排序

```
交互规则:
- 点击可排序列头 → 切换升序/降序
- 三态循环: 无排序 → 升序 → 降序 → 无排序
- 排序时 API 重新请求

视觉反馈:
- 列头显示排序图标
- 升序: ↑ 图标
- 降序: ↓ 图标
- 无排序: ↑↓ 双箭头图标（淡色）
```

### 3. 筛选

```
交互规则:
- 下拉选择 → 选择后立即触发筛选
- 搜索框 → 输入后 Enter 或点击搜索按钮触发
- 日期范围 → 选择后立即触发
- 重置 → 清空所有筛选条件

视觉反馈:
- 有筛选条件时，筛选栏背景变为浅蓝
- 重置按钮高亮显示
- 数据加载时显示 loading
```

### 4. 批量操作

```
交互规则:
- 选中 ≥1 条时，显示批量操作栏
- 选中 0 条时，批量操作栏隐藏
- 批量操作需要确认弹窗（危险操作）

批量操作栏内容:
- 显示已选中数量
- 批量分配角色、批量启用、批量禁用、批量删除按钮
```

## 表单交互

### 1. 输入验证

```
验证时机:
- 失焦验证 (推荐): 用户离开输入框时验证
- 提交验证: 点击提交时验证所有字段
- 实时验证: 仅用于格式化输入（如金额格式化）

错误显示:
- 输入框边框变红
- 输入框下方显示红色错误文字
- 不显示弹窗（避免打断用户）
```

### 2. 异步验证

```
使用场景:
- 邮箱唯一性检查
- 用户名唯一性检查

交互流程:
1. 用户输入邮箱
2. 失焦时触发异步验证
3. 输入框显示 loading 小图标
4. API 返回结果:
   - 唯一: 清除错误
   - 已存在: 显示错误「该邮箱已被使用」
```

### 3. 提交按钮状态

```
按钮状态变化:
- 默认: 可点击
- 点击后: 显示 loading spinner + 文字变为「提交中...」
- API 成功: Toast 成功提示 + 恢复按钮或跳转
- API 失败: Toast 错误提示 + 恢复按钮可再次点击
```

### 4. 表单草稿

```
场景: 长表单填写过程中用户可能离开页面

解决方案:
- 自动保存草稿到 localStorage
- 用户返回时提示「有未完成的表单，是否恢复？」
- 提交成功后清除草稿
```

## 权限树交互

### 1. 级联勾选

```
勾选父节点:
→ 自动勾选所有子节点
→ 向上更新父节点的半选/全选状态

取消父节点:
→ 自动取消所有子节点
→ 向上更新父节点的状态

勾选子节点:
→ 向上联动: 检查兄弟节点
  - 全部勾选: 父节点变为勾选
  - 部分勾选: 父节点变为半选
  - 全部取消: 父节点变为取消
```

### 2. 展开/折叠

```
默认状态:
- 有勾选子节点的模块默认展开
- 无勾选子节点的模块默认折叠

点击行为:
- 点击 [▼]/[▶] 图标展开/折叠
- 点击节点名称不触发展开/折叠
- 双击节点名称展开/折叠（可选）
```

### 3. 快捷操作

```
展开全部:
- 所有模块节点展开
- 不影响勾选状态

折叠全部:
- 所有模块节点折叠
- 不影响勾选状态

全选:
- 勾选所有节点
- 展开所有模块节点

取消全选:
- 取消所有节点勾选
```

## 弹窗交互

### 1. 打开弹窗

```
触发方式:
- 点击按钮打开
- API 错误时自动弹出（如删除确认）

Focus 管理:
- 弹窗打开时，focus 移入弹窗内第一个可交互元素
- 或 focus 移到关闭按钮
```

### 2. 关闭弹窗

```
关闭方式:
- 点击关闭按钮 [×]
- 点击取消按钮
- 按 Escape 键
- 点击遮罩层（可选，确认弹窗建议禁用）

Focus 恢复:
- 弹窗关闭后，focus 恢复到触发弹窗的元素
```

### 3. 弹窗动画

```
进入动画 (250ms):
- opacity: 0 → 1
- scale: 0.95 → 1
- translateY: -10px → 0

退出动画 (150ms):
- opacity: 1 → 0
- scale: 1 → 0.95
```

## 删除确认交互

### 1. 二次确认方式

| 删除类型 | 二次确认方式 |
|---------|-------------|
| **删除用户** | 输入用户名 |
| **删除角色** | 输入角色名称 |
| **批量删除** | 输入「DELETE」 |
| **禁用用户** | 仅确认弹窗，无输入 |

### 2. 输入确认流程

```
流程:
1. 弹窗显示删除对象信息
2. 显示输入框「请输入 xxx 名称以确认删除」
3. 用户输入确认文字
4. 输入正确时，「确认删除」按钮启用
5. 输入错误时，按钮禁用
6. 点击确认删除 → API 请求

视觉反馈:
- 输入正确: 输入框绿色边框
- 输入错误: 输入框灰色边框
```

## Toast 提示交互

### 1. Toast 显示规则

| 类型 | 显示时间 | 操作 |
|------|---------|-----|
| **Success** | 3 秒 | 自动消失 |
| **Info** | 4 秒 | 自动消失 |
| **Warning** | 5 秒 | 自动消失 + 可手动关闭 |
| **Error** | 5 秒 | 自动消失 + 可手动关闭 |
| **Undoable** | 5 秒 | 显示「撤销」按钮 |

### 2. Toast 位置

```
默认位置: 右上角
- 不遮挡主要内容
- 多个 Toast 垂直堆叠
- 最新 Toast 在最上方

移动端位置: 顶部居中
- 全宽显示
- 不遮挡底部操作按钮
```

### 3. Toast 动画

```
进入 (300ms):
- 从右侧滑入 + 淡入
- translateX: 20px → 0
- opacity: 0 → 1

退出 (200ms):
- 淡出
- opacity: 1 → 0
```

## 分页交互

### 1. 页码切换

```
点击页码:
- 当前页码高亮显示
- 其他页码显示为普通按钮
- 数据加载时显示 loading

点击上一页/下一页:
- 第一页时，「上一页」按钮禁用
- 最后页时，「下一页」按钮禁用
```

### 2. 每页条数

```
选择每页条数:
- 选择后立即触发数据重新加载
- 重置到第一页
```

## 搜索交互

### 1. 搜索触发

```
触发方式:
- 按 Enter 键
- 点击搜索图标按钮
- 输入后自动搜索（可选，需 debounce 300ms）

清空搜索:
- 点击输入框右侧 × 按钮
- 清空输入框 + 清除筛选结果
```

### 2. 搜索结果

```
无结果时:
- 显示空状态
- 提示「未找到匹配结果」
- 显示「重置筛选」按钮

有结果时:
- 显示结果数量
- 高亮匹配文字（可选）
```

## 拖拽交互

### 1. 菜单排序拖拽

```
拖拽流程:
1. 悬停行 → 显示拖拽手柄图标
2. 点击手柄 → 行进入拖拽状态
3. 拖动 → 显示拖拽预览
4. 放置 → 更新排序位置
5. API 保存排序

视觉反馈:
- 拖拽中: 行变为半透明 + 显示虚线框
- 可放置位置: 显示插入线
- 不可放置: 显示禁止图标
```

### 2. 拖拽阈值

```
防止误触发:
- 移动距离 > 5px 才开始拖拽
- 拖拽时间 > 200ms 才开始拖拽
```

## 无障碍交互

### 1. Focus 管理

```
规则:
- Tab 顺序与视觉顺序一致
- 弹窗打开时 focus 进入弹窗
- 弹窗关闭时 focus 返回触发元素
- Focus 状态清晰可见（2px 外环）
```

### 2. 屏幕阅读器支持

```
语义化标签:
- 按钮: aria-label 描述按钮功能
- 图标按钮: aria-label 必须有
- 弹窗: role="dialog" + aria-modal="true"
- Toast: aria-live="polite"
- 表格: 正确的 <th> + scope
- 表单: label + input 关联

状态变化通知:
- 加载完成: aria-live="polite" 通知
- 错误: aria-live="assertive" 通知
- 数量变化: aria-live="polite" 通知
```

## 撤销操作

### 1. Undo Toast

```
使用场景:
- 删除数据
- 移除角色
- 禁用用户

交互流程:
1. 用户执行删除操作
2. Toast 提示「已删除」+ 显示「撤销」按钮
3. 5 秒内用户点击「撤销」
4. 恢复删除的数据
5. 5 秒后自动消失，数据永久删除

Toast 样式:
┌────────────────────────────────────┐
│ ✓ 已删除用户 admin    [撤销]      │
└────────────────────────────────────┘
```

## Loading 状态

### 1. 组件 Loading

```
按钮 Loading:
- 显示 spinner 图标
- 文字变为「提交中...」或「加载中...」
- 禁用点击

表格 Loading:
- 显示骨架屏
- 或显示空白表格 + spinner

页面 Loading:
- 显示骨架屏（推荐）
- 或显示全屏 spinner
```

### 2. 骨架屏使用

```
使用场景:
- 表格数据加载
- 卡片数据加载
- 详情页数据加载

不使用场景:
- 按钮点击后（使用 spinner）
- 快速操作（< 300ms）
```

## 空状态处理

### 1. 数据空状态

```
显示内容:
- 空状态图标
- 描述文字「暂无数据」
- 操作按钮「新建」「导入」

位置:
- 表格/列表区域内居中
- 高度与数据填充时相当
```

### 2. 筛选无结果

```
显示内容:
- 搜索图标
- 描述文字「未找到匹配结果」
- 操作按钮「重置筛选」

与空数据区分:
- 空数据: 无任何数据
- 筛选无结果: 有数据但筛选后为空
```
